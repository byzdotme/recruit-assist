# 面试评价评分细则

总分 100 分。评分时以 JD 为主要基准，结合简历背景和面试记录中的实际回答证据。没有证据时不加分；证据不足时降低分数和置信度。

## 评分维度

| 维度 | 分值 | 评分说明 |
| --- | ---: | --- |
| 专业水平 | 50 | 与 JD 核心能力相关的知识深度、方法论、问题拆解、方案取舍、技术或业务判断、结果复盘能力。优先使用面试回答中的细节、追问表现和可验证案例。 |
| 项目经验 | 25 | 简历项目与 JD 的相关度、候选人在项目中的实际职责、复杂度、成果质量、真实性信号和面试中对项目细节的解释能力。 |
| 团队适配及综合素质 | 25 | 沟通表达、协作方式、责任边界、主动性、复盘意识、抗压和岗位动机。不得基于年龄、性别、婚育、民族、宗教等敏感或无关属性评分。 |

## 分数解释

| 总分区间 | 建议结论 |
| --- | --- |
| 85-100 | 强推荐进入下一轮或录用讨论 |
| 70-84 | 推荐，但需要针对关键风险继续验证 |
| 55-69 | 谨慎推进，仅在候选池不足或岗位要求可调整时考虑 |
| 0-54 | 不推荐 |

## 置信度

- 高：JD、简历和面试记录完整，关键问题有明确回答和追问证据。
- 中：主要信息完整，但部分关键能力只被间接覆盖或缺少追问。
- 低：面试记录过短、缺少关键回答、记录质量差或存在明显矛盾。

## 证据使用规则

- 面试回答优先作为评价依据；简历用于补充背景和识别需要复核的矛盾。
- 面试官主观评价可以作为线索，但必须回到候选人回答或可见事实上解释。
- 对明显错别字和口语化表达，可以按上下文修正；修正不确定时保留原意并标注“不确定”。
- 对未覆盖的信息不做负面推断，只说明证据不足或需要补充追问。
